# Roadmap 15 seances - ChlouhIN

## Principe general
- Une seance = un livrable visible.
- Windsurf construit.
- Open WebUI cadre, explique les fondamentaux et donne le bon prompt.
- On avance par micro-sprints.
- On ne saute pas d'etape tant que la precedente n'est pas verifiee.

## Seance 1 - Sprint 0A
Objectif : creer la base technique du projet.
Livrable : projet Next.js + TypeScript + Tailwind + shadcn/ui cree et lance en local.
L'eleve doit comprendre : Next.js = structure, TypeScript = securite, Tailwind = style, shadcn/ui = composants.

## Seance 2 - Sprint 0B
Objectif : versionner et publier la base du projet.
Livrable : git init, premier commit, repo GitHub connecte, projet pousse.
L'eleve doit comprendre : Git garde l'historique, GitHub partage et sauvegarde.

## Seance 3 - Sprint 0C
Objectif : mettre le projet en ligne.
Livrable : projet connecte a Vercel, URL publique fonctionnelle.
L'eleve doit comprendre : deployer = rendre visible le projet sur Internet.

## Seance 4 - Sprint 1A
Objectif : poser la structure des pages principales.
Livrable : home, map, login, dashboard, beth-habad/[slug], chaliah/[slug].
L'eleve doit comprendre : une app est un ensemble de routes/pages.

## Seance 5 - Sprint 1B
Objectif : construire la navigation commune.
Livrable : navbar simple et propre, visible sur les pages utiles.
L'eleve doit comprendre : la navigation organise l'experience utilisateur.

## Seance 6 - Sprint 1C
Objectif : transformer la home en debut de vrai produit.
Livrable : page d'accueil simple, lisible, avec proposition de valeur.
L'eleve doit comprendre : une home doit expliquer en 5 secondes ce que fait le produit.

## Seance 7 - Sprint 2A
Objectif : definir la memoire du produit.
Livrable : schema.sql avec les tables metier principales.
L'eleve doit comprendre : la base de donnees decrit ce que l'app retient.

## Seance 8 - Sprint 2B
Objectif : creer la base dans Supabase.
Livrable : tables creees, relations en place, verification dans Supabase.
L'eleve doit comprendre : schema -> execution -> tables reelles.

## Seance 9 - Sprint 2C
Objectif : connecter l'app a Supabase.
Livrable : client Supabase installe, variables d'environnement configurees, helpers crees.
L'eleve doit comprendre : l'app et la base doivent pouvoir se parler.

## Seance 10 - Sprint 3A
Objectif : mettre la porte d'entree du produit.
Livrable : pages login et signup, parcours de connexion simple.
L'eleve doit comprendre : authentification = savoir qui est l'utilisateur.

## Seance 11 - Sprint 3B
Objectif : proteger les zones privees.
Livrable : dashboard protege, logout fonctionnel, base de session stable.
L'eleve doit comprendre : une app decide qui voit quoi.

## Seance 12 - Sprint 4
Objectif : construire l'identite utilisateur.
Livrable : page profil publique + page edition du profil.
L'eleve doit comprendre : un profil est la fiche visible de l'utilisateur.

## Seance 13 - Sprint 5
Objectif : integrer les lieux et la carte.
Livrable : donnees Beth Habad + page map + fiches lieu basiques.
L'eleve doit comprendre : le produit relie des personnes et des lieux.

## Seance 14 - Sprint 6
Objectif : lier un Chaliah a un Beth Habad de maniere fiable.
Livrable : claim, file d'attente admin, validation simple.
L'eleve doit comprendre : la confiance produit passe par la verification.

## Seance 15 - Sprint 7
Objectif : consolider le produit et preparer la suite.
Livrable : demo propre du parcours complet, bugs critiques corriges, roadmap de suite.
L'eleve doit comprendre : un produit se termine par integration, verification et iteration.

## Regles de rythme
- Pas plus d'une idee neuve importante a la fois.
- Chaque seance se termine par une verification concrete.
- Si une etape casse, on ne continue pas tant qu'elle n'est pas comprise ou repartee.
- Si la classe va vite, on prend le sprint suivant, pas une variante decorative.
